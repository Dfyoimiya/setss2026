## 1. 项目背景与目标

SETSS 2026（Software Engineering Technology Symposium & Summit 2026）是一场国际学术会议，本系统为其官方网站及会议管理平台。

**系统目标**：

- 为参会者、作者、审稿人、组委会提供统一的线上服务平台
- 支持论文投稿、同行评审、注册报名等核心业务流程
- 展示会议信息，扩大会议影响力

---

## 2. 用户角色定义

| 角色 | 说明 |
|------|------|
| **访客（Guest）** | 未登录用户，可浏览公开信息 |
| **作者（Author）** | 注册用户，可投稿论文并跟踪审稿状态 |
| **审稿人（Reviewer）** | 由管理员指定，负责评审分配的论文 |
| **参会者（Attendee）** | 注册报名参加会议的用户 |
| **管理员（Admin）** | 会议组委会成员，管理全局数据与流程 |

> 一个用户可同时拥有多个角色（如作者同时也是参会者）。

## 3. 必须实现的功能（Must Have）

### M1 用户账户系统

#### M1.1 注册与登录

- **注册**：通过邮箱 + 密码创建账户，邮箱需唯一；密码后端哈希存储（bcrypt）
- **登录**：校验邮箱/密码，登录成功返回 JWT Token
- **登出**：客户端丢弃 Token（无状态 JWT）
- **密码找回**：通过注册邮箱发送重置链接

#### M1.2 个人信息管理

- 查看与修改姓名、所属机构、联系方式
- 修改密码（需验证旧密码）

---

### M2 会议信息展示

#### M2.1 首页

- 会议名称、主题、时间、地点展示
- 重要日期倒计时（投稿截止日、通知日、注册截止日、会议日期）
- 最新公告滚动展示

#### M2.2 公告与新闻

- 管理员发布/编辑/删除公告
- 访客可浏览所有公告，按时间倒序排列

#### M2.3 静态页面

- **Call for Papers**：征稿说明、投稿方向、格式要求
- **委员会页面**：大会主席、程序委员会成员介绍
- **联系我们**：组委会联系信息

---

### M3 论文投稿系统

#### M3.1 提交论文

- 作者填写：论文标题、摘要、关键词、研究领域（从预设列表选择）
- 上传论文 PDF 文件（限制大小，如 ≤20MB）
- 支持多位作者，指定通讯作者
- 提交后生成唯一投稿编号

#### M3.2 投稿管理

- 作者可查看自己所有投稿及当前状态
- 在截止日期前可修改投稿信息或更换文件
- 论文状态流转：`已提交 → 审稿中 → 已接受 / 已拒绝 / 需修改`

#### M3.3 作者通知

- 审稿结果确定后，系统通过邮件通知作者
- 论文被接受后，作者需上传最终版本（Camera-Ready）

---

### M4 同行评审系统

#### M4.1 审稿分配

- 管理员将论文分配给审稿人（每篇论文至少 2 位审稿人）
- 审稿人可查看分配给自己的待审论文列表

#### M4.2 提交审稿意见

- 审稿人提交评分（如 1–5 分）和详细文字评论
- 对作者的推荐决定：接受 / 拒绝 / 需要修改后再审
- 支持双盲评审（作者信息对审稿人不可见）

#### M4.3 审稿状态跟踪

- 管理员可查看所有论文的审稿进度
- 超时未提交时可向审稿人发送提醒

---

### M5 参会注册系统

#### M5.1 注册报名

- 用户选择参会类型（学生 / 普通 / 演讲者等）
- 填写个人信息（机构、职位、饮食偏好等）
- 提交后生成注册确认码

#### M5.2 注册状态管理

- 用户可查看自己的注册状态
- 管理员可查看、审核、导出注册名单

---

### M6 管理员后台

#### M6.1 用户管理

- 查看所有用户列表，支持搜索与筛选
- 修改用户角色（如指定某用户为审稿人）
- 禁用/启用账户

#### M6.2 投稿管理

- 查看所有投稿，按状态、时间、领域筛选
- 手动更改论文状态
- 批量导出投稿信息（CSV）

#### M6.3 系统配置

- 设置各关键日期（投稿截止、通知日期等）
- 管理研究领域/主题列表
- 发布/管理公告

## 4. 建议实现的功能（Should Have）

### S1 会议日程

- 管理员录入日程（时间、地点、演讲者、论文标题）
- 访客可按日期/时间查看完整日程
- 支持日程按 Track（专题） 筛选
- 提供日历订阅链接（.ics 格式）

### S2 演讲者页面

- 受邀主旨演讲者（Keynote Speaker）介绍页
- 展示照片、简介、演讲题目与摘要

### S3 邮件通知系统

- 重要操作触发邮件：注册确认、投稿确认、审稿结果通知、日程提醒
- 管理员可发送全员广播邮件

### S4 文件管理

- 安全存储上传的论文 PDF（对外不可直接访问）
- 审稿人通过鉴权接口获取论文文件
- 管理员可下载全量论文打包文件

### S5 搜索功能

- 公开论文摘要的全文搜索（会议结束后开放）
- 日程关键词搜索

### S6 响应式前端

- 移动端与桌面端均可正常浏览与操作
- 支持中英文双语切换（i18n）

## 5. 可选扩展功能（Could Have）

### C1 在线支付集成

- 注册报名时集成第三方支付（支付宝 / Stripe）
- 自动生成收据与发票

### C2 论文讨论区

- 审稿人与作者之间的匿名反馈通道（修改回复）
- 管理员可作为仲裁方参与

### C3 数据统计看板

- 管理员可视化投稿数量、领域分布、注册趋势等统计图表
- 审稿进度热力图

### C4 会议直播/录播集成

- 嵌入第三方直播链接（Zoom / YouTube）
- 录播视频归档页面

### C5 徽章与证书

- 参会者/演讲者电子证书自动生成与下载（PDF）

### C6 推荐审稿人

- 基于论文关键词和审稿人研究方向，智能推荐合适的审稿人（算法辅助）

## 6. 非功能性需求

### 6.1 安全性

- 所有密码不得明文存储，使用 bcrypt 哈希
- JWT 设置合理有效期（如 Access Token 2h，可配置）
- 上传文件类型校验，防止恶意文件
- API 关键操作需验证权限（RBAC）
- HTTPS 强制（生产环境）
- 防止 SQL 注入（ORM 参数化查询）、XSS（前端输出转义）

### 6.2 性能

- 首页静态资源加载 ≤ 3 秒（良好网络条件下）
- API 普通查询响应时间 ≤ 500ms
- 支持至少 200 个并发用户正常访问

### 6.3 可用性

- 关键操作（投稿、注册）表单提供明确的错误提示
- 操作结果提供成功/失败反馈
- 前端支持主流现代浏览器（Chrome、Firefox、Safari、Edge 最新版）

### 6.4 可维护性

- 后端代码遵循 Ruff 规范，无 lint 错误
- 所有 API 接口有 OpenAPI 文档（自动生成）
- 核心业务逻辑有单元测试覆盖
- 配置通过环境变量管理，不硬编码敏感信息

### 6.5 可部署性

- 提供 Docker Compose 一键启动开发环境
- 提供 Dockerfile 用于生产容器构建
- CI 通过后方可合并代码至 `main` 分支
